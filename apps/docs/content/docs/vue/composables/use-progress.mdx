title: useProgress

---

The `useProgress` composable is used to manipulate the progress state.

## Import

```tsx
import { useProgress } from '@lop-town/bprogress-...';
```

<Callout title="Note">
  Replace the `...` in the import with the library you're using. For example, if
  you use `@lop-town/bprogress-nuxt`, import by doing `from '@lop-town/bprogress-nuxt'`
</Callout>

## Usage

```tsx
const {
  start,
  stop,
  inc,
  dec,
  set,
  pause,
  resume,
  setOptions,
  getOptions,
  isAutoStopDisabled,
  disableAutoStop,
  enableAutoStop,
} = useProgress();
```

## Methods

### start

Start the progress bar.

```tsx
start();
// or
start(0.3, 1000);
```

### stop

Stop the progress bar.

```tsx
stop();
// or
stop(1000);
```

### inc

Increase the progress bar (between 0 and 1).

```tsx
inc(0.1);
```

### dec

Decrease the progress bar (between 0 and 1).

```tsx
dec(0.1);
```

### set

Set the progress bar (between 0 and 1).

```tsx
set(0.5);
```

### pause

Pause the progress bar.

```tsx
pause();
```

### resume

Resume the progress bar.

```tsx
resume();
```

### setOptions

Set the progress bar options.

```tsx
setOptions({ showSpinner: false });
// or
setOptions((options) => ({ ...options, showSpinner: true }));
```

### getOptions

Get the progress bar options.

```tsx
const options = getOptions();
```

### isAutoStopDisabled

Check if the auto stop is disabled.

```tsx
const isDisabled = isAutoStopDisabled.value;
```

<Callout title="Note" type="info">
  When you navigate between pages, the stop is played automatically, so if you
  don't want the progress to stop by itself when you manipulate the URL, this
  ref is useful.
</Callout>

### disableAutoStop

Disable the auto stop.

```tsx
disableAutoStop();
```

### enableAutoStop

Enable the auto stop.

```tsx
enableAutoStop();
```
